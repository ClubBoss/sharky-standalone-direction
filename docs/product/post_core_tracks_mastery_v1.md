# Post-Core Tracks + Mastery v1

Status: Policy only, deferred implementation.

## 1) Goal
- Add post-core specialization depth without duplicating the full core curriculum.
- Keep deterministic operation and small reversible delivery.

## 2) Track options
- Cash Mastery
- Tournament Mastery (MTT)
- Not sure yet (Mixed playlist) as fallback only

Policy note:
- Mixed is not a third full track. It is a balanced fallback playlist.

## 3) Transition Explainer (60-120s)
Explain, then ask for track selection:
- Variance profile (cash usually steadier, mtt usually higher swings)
- Time structure (cash flexible sessions, mtt stage pressure)
- Skill emphasis (deep stack and sizing vs ICM and pressure)

If dismissed:
- Default to mixed.

## 4) Track structure v1
- Each track uses 3-5 post-core chapters (not 10 new worlds).
- Core Levels 0..9 remain shared and unchanged.

Example Cash chapters:
- Deep stacks and sizing norms
- Multiway frequency and value/protection
- Pool exploit patterns and bankroll discipline

Example MTT chapters:
- ICM deepening
- Bubble and final-table pressure
- Short and mid stack adaptation

## 5) Retention loop (no infinite content)
- Mastery tiers: Learn, Prove, Speed
- Leaks resurfacing queue: deterministic weak-area replay
- Daily rhythm remains deterministic and contract-testable

## 6) Anti-drift rules
- No early branching before Core completion.
- No schema changes for track v1.
- No new drill kinds for track v1.
- No random ordering in playlists.
- Keep code and content changes reversible and small.
