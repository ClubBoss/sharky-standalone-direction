## Scope / non-scope

coverage benchmark and compares it against Sharky's current active SSOT,
first-week proof notes, and authored content evidence.

glossary, UI, routes, telemetry, Modern Table, queue lifecycle, monetization,
screenshots, tests, or generated outputs. It does not copy Runout content,
assets, layouts, lesson bodies, or proprietary implementation. It does not turn
Runout taxonomy into Sharky roadmap authority.

## Executive verdict

Runout is clearly broader and more commercially packaged. Sharky is currently
more differentiated where it matters for first-week trust: deterministic
learning causality.

Runout appears to sell a complete poker improvement ecosystem. Sharky should
not respond by becoming a textbook clone. Sharky's near-term product wedge is:

`choice -> visible table signal -> why -> repair/recheck proof`

The W5 board-texture same-signal fix closes the most immediate content-depth
blocker for first-week proof review. The next highest-EV Sharky repo wave is
external proof packaging, not another content expansion.

## Runout structure summary

Runout's installed map presents a broad curriculum hierarchy:

- Preflop.
- Betting & Aggression.
- Defense & Response.
- Math & Theory.
- Hand Analysis & Decision Making.
- Positional & Situational.
- Tournament Strategy.

It covers visible skills such as preflop hand selection, 3-betting, facing
3-bets, blind defense, c-betting, facing c-bets, check-raising, probe betting,
bet sizing, value betting, bluffing, pot odds, implied odds, SPR, outs/equity,
hand reading, blockers, position play, multiway pots, drawing hands, and ICM.

Static extraction also found commercial ecosystem signals: onboarding,
paywall/subscription infrastructure, progress/mastery/rank surfaces,
daily-session/review/recalibration strings, and visual-key families.

## Sharky current first-week proof summary

Sharky's accepted first-week claim is narrower:

- Placement / Welcome gets the learner to one small real decision.
- The W1 runner shows a table decision.
- Feedback explains a visible table signal.
- Repair focus/result/session proof can appear in the Act0 flow.
- Review/Profile can show proof without becoming dashboards.
- W6 range-bucket and W5 board-texture have deterministic repair/recheck seams.

The current accepted proof is ready for external first-week review if the claim
stays scoped. It is not a claim of full content breadth, premium readiness, or
completed queue resolution.

## Coverage comparison

Runout has broader taxonomy coverage. It exposes many skills and chapter rows
across preflop, postflop aggression, defense, math, hand analysis, position,
multiway, draws, and tournaments.

Sharky has enough first-week content and proof to demonstrate the core learning
loop, but not enough to claim Runout-like breadth. This is acceptable because
Sharky's positioning is not textbook breadth; it is table-first repair proof.

Classification:

- Runout breadth: `valuable_after_external_review`.
- Sharky first-week proof: `already_stronger_in_sharky`.
- Broad parity expansion: `do_not_copy`.

## Sequencing comparison

Runout's likely first path starts with onboarding / first-session / skills
library, then a Preflop path with beginner, intermediate, and advanced chapters.
That sequence can create a strong course-like impression, but it also introduces
heavier poker vocabulary early.

Sharky's first-week route starts smaller: table anchors, legal action, street
flow, one real decision, feedback, and repair proof. This is safer for beginner
cognitive load and better aligned with the current product thesis.

Classification:

- Runout sequencing as benchmark: `valuable_after_external_review`.
- Sharky first-session cognitive load: `already_stronger_in_sharky`.

## Drill density / repetition comparison

Runout's visible chapter item counts create an impression of large drill and
lesson density. The map is broad enough to make Sharky look narrower if Sharky
shows only raw content count.

Sharky's strongest repetition evidence is narrower but causal:

- Act0 first-week proof.
- W6 range-bucket same-signal expansion.
- W5 board-texture same-signal expansion.
- Review queue display for supported session-drill rechecks.

The W5 same-signal fix closes the immediate first-week proof weakness. Further
content expansion should wait until after external proof packaging identifies a
specific trust gap.

Classification:

- W5 same-signal: `already_stronger_in_sharky` for repair-backed proof.
- Broad density: `defer_until_learning_depth_phase`.

## Feedback / repair comparison

Runout has static evidence for session review, skill drills, mastery, radar,
recalibration, ratings, ranks, and progress surfaces. The installed research
does not prove causal repair behavior.

Sharky has the stronger current claim: deterministic causality from missed
signal to repair/recheck opportunity.

Classification: `already_stronger_in_sharky`.

Recommendation: lead the external review with Sharky's causal proof rather than
apologizing for narrower breadth.

## Visual explanation comparison

Runout likely has more reusable visual explainers. The static report found
`lessons.visuals.*` key families for blockers, ranges, bet sizing, c-betting,
draw equity, SPR/stack depth, street progression, action diagrams, and
value/bluff structure.

Sharky should learn the job-to-be-done: visual explanations should make a
decision signal concrete. It should not copy Runout visuals, layouts, assets,
or chart patterns. Any Sharky visual work should remain tied to the table-signal
proof loop.

Classification: `valuable_after_external_review`.

## Commercial packaging comparison

Runout is stronger in commercial packaging: paywall assets, billing,
subscription/trial strings, progress theatre, daily/session/review surfaces,
and ecosystem breadth.

Sharky is stronger in trust timing: value before paywall. The W1-W4 free /
W5+ future paid-depth boundary remains locked. Runout's commercial packaging is
a later benchmark, not permission to start paywall work now.

Classification:

- Packaging gap: `valuable_after_external_review`.
- Paywall implementation now: `not_current_scope`.
- Runout artifact imitation: `do_not_copy`.

## Beginner cognitive-load comparison

Runout's Preflop path reaches range charts, 3-betting, combinatorics, SPR, and
multiway shifts within one visible skill family. That can signal seriousness,
but it can also overload a newer learner.

Sharky's first-week route keeps the opening load lower and proves one concrete
decision. This is an advantage as long as the proof packet communicates it
clearly.

Classification: `already_stronger_in_sharky`.

## What Sharky should learn

1. Breadth helps commercial confidence, but only after first value is obvious.
2. Visual explainers are useful when they clarify a decision signal.
3. A dense skill/chapter map can make an app feel serious.
4. Commercial packaging needs a clear value narrative before paywall.
5. Progress surfaces should support learning proof, not replace it.

## What Sharky should not copy

- Runout lesson bodies, assets, layouts, icons, charts, or paywall composition.
- Broad category taxonomy as Sharky roadmap authority.
- Early advanced vocabulary as first-session experience.
- Dashboard/progress theatre before learner improvement is visible.
- Subscription/trial pressure before Sharky has shown value.
- Generic GTO/course breadth if it competes with table-signal repair proof.

## Top 5 high-EV implications

1. Package the current first-week proof for external review before adding more
   content.
2. Keep the narrow proof claim: `choice -> table signal -> why -> repair`.
3. Use Runout's breadth as a later content-depth audit checklist, not a roadmap.
4. Treat visual explainers as future signal-clarity tools, not Modern Table
   polish.
5. Keep monetization deferred until proof packaging and trust evidence are
   reviewed.
