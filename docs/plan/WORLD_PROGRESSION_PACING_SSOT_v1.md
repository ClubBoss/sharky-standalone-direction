# World Progression Pacing SSOT v1

Purpose:

- define the first canonical pacing and rhythm truth layer for world / module / session growth
- keep future expansion governed by one explicit progression contract instead of local intuition
- stay bounded to pacing only, not prerequisites, release gating, monetization, or runtime implementation

## Governs Now

This SSOT governs:

- the intended learning rhythm inside one world block
- how quickly local difficulty may rise
- how many genuinely new concepts may be introduced in one bounded stretch
- when recap, review, or checkpoint interruption is required
- acceptable cognitive density for future world / module / session planning

This SSOT does not yet govern:

- explicit prerequisite graph logic
- anti-boredom / novelty scheduling as a full system
- release gating rules
- runtime routing or host implementation
- pricing, entitlement, or monetization strategy

## Canonical Local Rhythm

Default rhythm inside a bounded world slice:

1. intro / anchor
2. guided discrimination
3. applied choice
4. short mixed reinforcement
5. checkpoint or recap

Operational rule:

- a local world block should not stay in pure new-concept teaching for more than 3 consecutive sessions without either:
  - recap / review interruption
  - or a bounded checkpoint

## New-Concept Introduction Limits

Per local block or short session stretch:

- preferred: 1 genuinely new concept
- allowed maximum: 2 tightly related new concepts
- disallowed: 3 or more unrelated new concepts introduced as first-time material in one local stretch

Interpretation:

- a “new concept” means a new decision idea, comparison frame, or reasoning primitive
- new examples of the same concept do not count as additional new concepts
- a new interaction shell alone does not justify extra concept load

## Difficulty Rise Rule

Difficulty may rise only one step at a time along these axes:

- recognition -> classification
- classification -> action choice
- single-step -> short chain
- guided visibility -> reduced guidance
- isolated concept -> bounded mixed application

Operational rule:

- one local stretch may escalate on one major axis strongly
- if two axes rise together, each rise must stay mild
- do not stack a sharp concept jump, format jump, and reduced guidance jump at once

## Cognitive Density Limits

In one bounded session or cluster:

- preferred working-set size: 1 core target + 1 supporting cue
- allowed upper bound: 1 core target + 2 tightly coupled cues
- avoid:
  - multiple simultaneous hand-taxonomy jumps
  - multiple new exception rules
  - mixed review before the base concept is stable

Practical meaning:

- learners should usually know what the single main job is
- secondary details may support the job but should not compete with it

## Required Interruptions

Review / recap / checkpoint is required when any of these become true:

- 3 consecutive new-concept sessions have occurred
- a major difficulty-axis rise just completed
- the next step would require combining 2 previously separate ideas
- a world slice is about to switch from guided work to mixed or freer application

Allowed interruption forms:

- short recap
- compact review queue
- bounded checkpoint
- short mixed reinforcement using already-taught concepts only

## Variety Rule

Variety should interrupt monotony, but only after stability exists.

Operational rule:

- after 2 to 4 sessions of the same cognitive job, insert one of:
  - a recap
  - a mixed reinforcement slice
  - a nearby-family application slice

Do not use variety to hide unstable understanding.

## Checkpoint Rule

A checkpoint is appropriate when:

- the learner has already seen the concept in guided and applied form
- the next planned growth step depends on stable recall
- failure should signal reinforcement, not concept surprise

A checkpoint should not:

- introduce a brand-new concept
- combine more than 2 already-learned ideas for the first time
- serve as a disguised difficulty spike

## Mixed Review Rule

Mixed review is allowed only when:

- each included concept already has a stable single-concept form
- the mix tests discrimination or transfer, not first exposure

Default limit:

- a mixed slice should usually combine 2 families
- 3 families is the upper bound and should be reserved for later stable worlds or capstones

## Pacing Decision Table

| Situation | Required pacing action |
| --- | --- |
| one new concept just introduced | follow with guided or applied repetition before adding another new concept |
| two related concepts introduced in one local stretch | next step must recap or apply, not add a third concept |
| three consecutive sessions without interruption | insert recap, review, or checkpoint |
| upcoming step needs transfer across families | use bounded mixed reinforcement first |
| upcoming step requires lower guidance and new reasoning at once | split the rise into two steps |

## Canonical Preservation Rules

- prefer boredom prevention through bounded variety, not through uncontrolled concept sprawl
- prefer explicit recap over silent assumption that prior learning stuck
- prefer short reinforcement before capstone mixing
- preserve room for future worlds to escalate by not exhausting too many concept axes early

## Out of Scope For Later SSOTs

These belong to later documents:

- prerequisite dependency matrix
- release gate / content factory enforcement
- boredom-management and novelty cadence as a standalone system
- monetization pacing
- world-count or season-count planning
