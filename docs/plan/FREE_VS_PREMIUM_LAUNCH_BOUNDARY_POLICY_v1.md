# Free Vs Premium Launch Boundary Policy v1

Status: ACTIVE
Last updated: 2026-05-06

## Purpose

Define the canonical launch boundary between free and premium for the current
Sharky product route.

This document is narrower than the app-wide monetization guideline.
It answers one launch-critical question:

- what must stay free at launch
- what premium should unlock at launch
- what must not be monetized too early

## Core Decision

Launch monetization should use:

1. a real free trust-building route
2. premium as deeper sharpening and deeper route access
3. trial only after value proof

Not:

1. hard early route rescue gating
2. pressure before first useful learning loop
3. fake free value with the real product hidden behind the first wall

## Launch Product Shape

At launch, the user should experience the product as:

1. a credible free learning path
2. a believable reason to return
3. a visible deeper layer worth paying for

The user should not experience it as:

1. a teaser that blocks usefulness
2. a paywall-first app
3. a one-day novelty with no deeper reason to subscribe

## What Must Stay Free At Launch

The free path must include enough real value to establish trust.

### Free obligations

1. first-user understanding of the product
2. onboarding / placement / trust-primer completion
3. a real first-win learning loop
4. access to a bounded but meaningful portion of `Volume I`
5. at least one believable daily-return reason
6. enough Review / repair truth that the app feels useful, not decorative

### Free should prove

1. this app teaches real poker improvement
2. this app is worth returning to
3. this app has more depth than the currently free slice

## What Premium Should Unlock At Launch

Premium should unlock deeper usefulness, not basic credibility.

### Premium launch value

1. fuller route access across the active launch surface
2. stronger replay and mastery depth
3. richer weak-spot resurfacing and repair usefulness
4. premium-target worlds or paths that extend the active route honestly
5. better long-horizon reasons to keep using the app after the first wins

### Premium feeling

Premium should feel like:

1. sharper
2. deeper
3. more personal
4. more durable

Premium should not feel like:

1. finally usable
2. unblocked basics
3. a tax on trust

## Launch Boundary Rule For Volume I

`Volume I` is the launch-critical route.
Its free/premium boundary must be honest and strategically useful.

Rule:

1. do not give away so little that trust never forms
2. do not give away so much that premium has no sharpening role
3. do not use the boundary to create artificial grind

## Trial Timing

Launch trial policy:

1. no trial before meaningful value is visible
2. no trial as a substitute for basic free trust
3. trial after placement or another real proof moment is acceptable
4. trial must preview deeper sharpening, not rescue a blocked app

## Hard Disallow List

At launch, do not:

1. block the first useful learning loop behind premium
2. block basic route comprehension behind premium
3. place hard monetization pressure before habit credibility
4. use fake scarcity, guilt, or punishment loops
5. rely on monetization because the route itself is still weak

## Retention Relationship

The free path should be strong enough to create:

1. trust
2. a first improvement signal
3. a second-session reason

Premium should then create:

1. longer retention
2. stronger sharpening
3. more personal usefulness
4. later subscription durability

## Decision Table

| Launch question | Default answer |
| --- | --- |
| Should the first useful loop be premium-only? | No |
| Should trial appear before trust/value proof? | No |
| Should premium sell deeper route and sharpening? | Yes |
| Should the free path be complete enough to build trust? | Yes |
| Should monetization pressure wait until habit/value credibility? | Yes |

## Current Launch Interpretation

For the current app route:

1. `W1-W12` remains the canonical launch route shape
2. the free launch slice must still feel real and usable
3. premium should be framed as deeper route access and stronger long-horizon
   sharpening
4. `W13+` are not the launch monetization answer; they are later expansion value

## Bottom Line

The best launch boundary is:

1. generous enough to build trust
2. bounded enough to preserve paid depth
3. explicit enough to stop future drift

That is stronger than either extreme:

1. weak teaser free path
2. almost-all-free path with no meaningful premium role
