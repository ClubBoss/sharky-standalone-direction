# Progression / Prerequisite Matrix v1

Status: SSOT-lite planning canon  
Scope: sequencing, prerequisites, anti-jump guidance for future world/node fill  
Use: reference for rollout planning, future guards, and bounded slice decisions

## Purpose

Coverage completeness is not enough on its own. A learning path can still fail if skills appear before their prerequisites, if worlds jump in difficulty too fast, or if modes are introduced without a bridge.

This matrix records the current canonical progression guidance for:

- skill-family prerequisite order
- mode-family prerequisite order where it matters
- earliest safe world anchors
- anti-jump warnings for future fill work

## Core Progression Rules

1. Foundational recognition before decision pressure.
   Learners should identify table elements, roles, and order before being asked to make richer strategic choices.

2. Simple mental models before derived judgment.
   Learners should know what a concept is before they are asked to compare, classify, size, or justify it.

3. Single-step choice before multi-step reasoning.
   A mode that asks for one action or one comparison should come before chains, multi-street logic, or mixed challenges.

4. Purpose before optimization.
   Learners should understand what an action or size is trying to accomplish before being asked to choose among close variants.

5. Concrete counting before abstract intuition.
   Outs and improvement counting should come before equity or pot-odds intuition.

6. World rhythm should default to:
   intro -> practice -> apply -> review

7. New mode families should enter through a bounded pilot or clearly scaffolded node family, not through broad simultaneous rollout.

## World Rhythm Principle

Each world or major node family should normally express a simple rhythm:

1. Intro
   Define the concept or target plainly.
2. Practice
   Use low-complexity reps in one or two strong mode families.
3. Apply
   Ask for a slightly richer choice, comparison, or contextual use.
4. Review
   Revisit mistakes, reinforce the compact rule, and stabilize recall.

This rhythm does not require four separate screens every time. It does require that rollout avoids cold jumps from zero context to abstract decision pressure.

## Practical Anti-Jump Warnings

- Do not place equity / pot-odds intuition before outs / improvement counting exists.
- Do not place sizing optimization before bet purpose exists.
- Do not place range-construction-lite content before hand comparison and preflop framework basics exist.
- Do not introduce multi-step chain before simple action choice is already stable.
- Do not rely on speed or mixed modes to carry first exposure.
- Do not treat architectural mention as proof of safe sequencing.

## Near-Term Implication

The project should continue to protect World 1 as the onboarding and literacy base, treat World 2-3 as the most likely next structured bridge into hand strength, action choice, and bet-purpose work, and avoid adding advanced intuition or chain-heavy content before those bridges are explicit.
