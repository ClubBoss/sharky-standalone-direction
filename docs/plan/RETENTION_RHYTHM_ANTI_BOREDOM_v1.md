# Retention Rhythm / Anti-Boredom v1

Purpose:

- define the first canonical long-horizon retention and anti-boredom truth layer for future world / module / session expansion
- keep future expansion governed by explicit variation and reinforcement rhythm instead of local intuition
- stay bounded to retention rhythm only, not monetization timing, runtime personalization, energy systems, or full gamification

## Governs Now

This SSOT governs:

- acceptable repetition limits before variation is required
- when nearby-family variation should interrupt monotony
- how review should be spaced in principle
- the intended confidence-step vs challenge-step rhythm
- how to avoid “endless same drill” feel while preserving stability

This SSOT does not yet govern:

- runtime personalization or adaptive scheduling
- streaks, rewards, economy, or monetization timing
- session energy budget
- mastery-map visualization
- mascot / progression layer behavior

## Core Retention Principle

Retention should come from:

- stable understanding
- bounded variation
- visible return to earlier skills
- challenge introduced after confidence, not instead of confidence

It should not come from:

- endless identical reps
- novelty for novelty’s sake
- fake urgency
- difficulty spikes used as entertainment

## Repetition Limits

Acceptable repetition before variation is required:

- same exact mode/family job:
  - preferred run length: 2 to 4 reps/sessions
  - soft upper bound: 5
- same exact prompt shape with minimal variation:
  - preferred run length: 2 to 3
  - hard stop before broad rollout: do not let the path feel like one unchanged rep loop

Operational rule:

- once a learner has seen the same cognitive job 2 to 4 times in a row, the next step should usually vary one of:
  - example surface
  - nearby-family use
  - review framing
  - apply/checkpoint framing

## Variation Rule

Variation should be introduced in this order:

1. same concept, new example
2. same family, slightly richer context
3. nearby-family application
4. bounded mixed review

Avoid skipping directly from:

- same rep loop
to
- broad mixed challenge

Variation must preserve the main job.

## Nearby-Family Variation Rule

Nearby-family variation is allowed when:

- the original family is already stable enough to avoid concept collapse
- the nearby family reinforces the same broader skill cluster
- the learner still has one dominant job, not multiple competing jobs

Examples:

- recognition -> action choice
- showdown comparison -> outs/counting
- action choice -> sizing purpose

Do not use nearby-family variation if it silently introduces a new prerequisite boundary.

## Review Spacing Principles

Review should be spaced by instability and distance, not by arbitrary repetition quotas alone.

Required review triggers:

- after a checkpoint or challenge step
- after a family has been absent long enough that recall may weaken
- after a local concept cluster completes
- after error-prone or fragile understanding becomes visible

Preferred review spacing pattern:

- immediate short reinforcement
- later bounded resurfacing
- eventual mixed reappearance in a different but related context

## Confidence-Step vs Challenge-Step Rhythm

Default rhythm:

- confidence step
- confidence step or mild apply step
- challenge step
- reinforcement or review step

Operational rule:

- challenge steps should not outnumber stabilizing steps in an immature family
- a challenge step should usually be followed by reinforcement, recap, or a cleaner reattempt path

Interpretation:

- confidence step:
  - improves fluency, recognition, and “I know what this is”
- challenge step:
  - tests transfer, richer discrimination, or reduced guidance

## Endless-Same-Drill Prevention Rule

A path starts to feel grindy when too many of these stack:

- same family
- same prompt structure
- same difficulty band
- same correction rhythm
- no visible progression or recap change

Required response:

- change one structural dimension before expanding the run further

Allowed structural changes:

- add recap
- add apply framing
- switch to nearby-family reinforcement
- insert mixed review
- use checkpoint to mark progression

## Stability Preservation Rule

Anti-boredom measures must not destroy stability.

Do not:

- insert broad novelty before the core concept is stable
- rotate families so often that the learner never consolidates
- use cosmetic variation as a substitute for meaningful progression

If forced to choose:

- prefer slight boredom over conceptual instability
- then relieve boredom through bounded structured variation

## Long-Horizon Rhythm

Across a larger world/module stretch, a healthy rhythm should usually include:

- fresh concept introduction
- short repetition run
- applied use
- recap or checkpoint
- resurfacing later in a different local context

This means retention is not one review event.
It is repeated reappearance with controlled variation.

## Decision Table

| Situation | Required retention action |
| --- | --- |
| same family repeated 2 to 4 times | insert bounded variation soon |
| challenge step just occurred | follow with reinforcement, recap, or cleaner reattempt |
| concept cluster just closed | add recap, checkpoint, or later resurfacing plan |
| family absent for a meaningful stretch | resurface with compact review before assuming transfer |
| path feels repetitive but family is still unstable | vary example/context mildly, not the whole cognitive job |

## Governing Use Rule

Before extending a path, ask:

1. Has this family/job repeated long enough to risk monotony?
2. Is the learner ready for variation without losing the core concept?
3. Should the next step be confidence, challenge, or review?
4. Does the proposed variation preserve the same broader skill cluster?
5. Would this step feel like progress, not just more of the same?

If those answers are unclear, do not broaden the repetition run further.
