## Core Decision

The product should monetize as a depth-and-sharpening system, not as an
early-access rescue gate.

That means:

1. free value must be real
2. the first habit loop must be believable before strong monetization pressure
3. subscription should feel like deeper sharpening, not permission to begin
4. long-term retention should come from mastery, resurfacing, and later route
   depth, not fake urgency

## Strategy Hierarchy

When monetization and retention decisions conflict, use this order:

1. learner trust
2. real learning value
3. return usefulness
4. retention durability
5. monetization pressure

This means monetization may amplify a strong product.
It must not compensate for a weak one.

## Product Model Across The Full App

### Layer 1. Free Core Value

The learner must be able to:

1. understand what the product is
2. complete a meaningful first-win loop
3. see that the app improves real play
4. return at least a few times for useful practice or repair

Free should not mean "empty teaser."
Free should mean "credible value with honest boundaries."

### Layer 2. Premium Sharpening

Premium should unlock deeper value, such as:

1. fuller route access across the active product surface
2. stronger daily best-next-action depth
3. richer repair and weak-spot resurfacing
4. mastery-tier progression beyond the first pass
5. denser later-world and later-volume route depth

Premium should feel like:

- more precise
- more personal
- more durable

It should not feel like:

- finally usable
- artificially unblocked
- pressure-first

### Layer 3. Later Expansion Value

The strongest long-term subscription value should come from:

1. Volume II strategic depth
2. Volume III specialization and pro layer
3. post-W12 specialization gateway
4. deeper mastery tiers
5. personal crucible / weak-spot resurfacing

This is what creates durable subscription logic without infinite beginner
content inflation.

## Canonical Retention Engine

The strongest long-term return loop is:

learn -> notice improvement -> prove it -> repair weakness -> return

This is better than:

1. streak guilt
2. noisy mascot loops
3. random challenge churn
4. aggressive paywall prompting

## Route-Aware Monetization Shape

### Volume I

`W1-W12` is the launch-critical shared route.

Monetization role:

1. prove value
2. prove return usefulness
3. prove that the product has more depth than a one-week novelty app

Volume I should not be stretched artificially only to increase subscription
duration.
It should be complete, strong, and honest.

### Volume II

`W13-W24` is the first major paid-depth expansion candidate.

Monetization role:

1. extend serious practical learning
2. make the subscription feel deeper, not wider only
3. reward returning learners with visible new strategic depth

### Volume III

`W25-W36` is the specialization and pro layer.

Monetization role:

1. become the strongest long-term advanced value layer
2. support format-specific retention and later high-LTV users
3. keep serious learners inside the ecosystem for months, not days

## Specialization And Monetization

Cash / Tournament specialization is one of the strongest later monetization and
retention events in the whole product.

Rule:

1. do not hard-fork early
2. use post-W12 specialization as a major value and identity moment
3. let subscription benefit from deeper recommended-track sharpening later

This makes specialization:

- meaningful
- earned
- high-value

instead of:

- premature
- confusing
- duplicative

## Free vs Premium Boundary

### Free must include

1. a real first-win route
2. a believable daily-return reason
3. enough visible learning progress to build trust
4. honest route boundaries

### Premium should own

1. deeper route access
2. stronger replay / mastery value
3. more personal resurfacing and repair depth
4. later-volume and later-specialization depth
5. stronger long-horizon reasons to stay subscribed

### Premium should not own too early

1. the first useful learning loop
2. basic trust in the product
3. basic route comprehension
4. the right to feel any value at all

## Trial Policy

Trial should be tied to a meaningful proof moment, not first touch.

Default rule:

1. no hard first-session monetization
2. trial after visible value and learner trust
3. trial should open deeper sharpening, not rescue a blocked product

Current best fit in the app:

- no trial before placement, first hand, first feedback, first completion, or
  the soft premium preview
- no trial inside the soft premium preview
- contextual 7-day trial at a future W5 locked-depth/paywall attempt, or later
  high-intent proof, after commerce safety is production-ready
- D2 return, preview-clicker, dynamic behavior, repair-depth, and
  analytics/leak-profile trial timing are future experiments, not launch
  default

### Trial job

Trial should answer:

- what does the deeper layer feel like
- why would I keep this
- how much more personal or sharpening does premium become

It should not answer:

- can I finally use the app

## Subscription Packaging Preference

This guideline does not set exact prices.
It does set the preferred packaging logic.

Recommended commercial shape:

1. annual as the best-value default
2. monthly as the lower-friction entry
3. no early dependence on weekly urgency traps

Reason:

1. the product is built for months-long improvement, not one-week novelty
2. annual aligns best with mastery, later volumes, and specialization depth
3. monthly remains useful for cautious or lower-commitment entrants

## Strongest Retention Drivers

Rank retention levers in this order:

1. personal weak-spot usefulness
2. mastery tiers and visible sharpening
3. daily best-next-action clarity
4. later route depth
5. specialization identity
6. light streak / achievement / progress visibility
7. Sharky emotional warmth

If a lower-ranked lever becomes louder than a higher-ranked one, the product is
drifting toward shallow retention.

## Anti-Churn Rules

Do not:

1. place monetization friction before a believable first-win loop
2. rely on pressure because route depth is still weak
3. make free users feel tricked or partially locked out before understanding
   value
4. use streak loss, guilt, or fake scarcity as the core return engine
5. make premium louder than usefulness

## Runtime Truth We Keep From Older Monetization Docs

Preserve these older strengths:

1. deterministic entitlement precedence
2. restore correctness
3. trial eligibility clarity
4. status-honest premium / trial messaging
5. stable routing under non-entitled, trial, premium, and restored states

These remain runtime obligations.
They are not the full monetization strategy by themselves.

## Practical Decision Table

| Product state | Monetization / retention decision |
| --- | --- |
| value is still weak or thin | improve content/product first, not offers |
| value exists but return habit is weak | strengthen Review / mastery / daily usefulness before louder monetization |
| value and habit are visible | allow value-first premium preview |
| Volume I is strong and later depth is believable | use subscription as depth expansion |
| post-W12 specialization becomes real | use specialization as a major retention and upsell event |

## Governing Use Rule

Before approving a monetization or retention proposal, ask:

1. does this increase real learner value or only pressure
2. does this strengthen durable return usefulness
3. does this preserve trust
4. is this aligned to route stage: Volume I, II, or III
5. is this selling depth, sharpening, or specialization rather than rescuing a
   weak free path

If those answers are weak, do not ship the proposal yet.

## Bottom Line

The best business shape for Sharky is:

1. strong free trust-building route
2. premium as deeper sharpening
3. retention through mastery and personal usefulness
4. later volumes and specialization as major subscription value multipliers
5. runtime entitlement discipline preserved underneath

That is the strongest combined path for:

1. profit
2. retention
3. trust
4. long-term learner value

without collapsing into either:

1. weak freemium fluff
2. early paywall pressure
3. noisy fake gamification
