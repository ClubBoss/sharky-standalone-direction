## Goal

Answer one question honestly:

Does a real first-time learner understand what this app is, trust the first
start, reach one useful action quickly, and feel at least one real early
improvement?

## Scope

In scope:

- first-time learner only
- first session only
- active Act0 route only
- `Placement -> Home -> first useful hand -> Result -> back to Home`

Out of scope:

- dormant systems
- account systems that do not exist yet
- deep curriculum exploration
- advanced poker users optimizing edge cases

## Proof Set

Minimum:

- `2` runs if both are clean and consistent
- `3` runs if signals split or one run is ambiguous

Use participants who:

- are new to poker or near-beginner
- have not already learned this product route
- can speak their thoughts out loud in simple terms

Do not use:

- builders of the product
- people who already know the route well
- players who can brute-force the route from prior poker knowledge

## Moderator Setup

Before the run:

1. Reset to a clean first-start state.
2. Use one admitted device class at a time.
3. Start screen recording if available.
4. Tell the participant:
   - "Please think out loud."
   - "There are no right answers about the app."
   - "If something is confusing, say it immediately."

Do not explain the product before the run beyond this:

"This is a poker learning app for beginners. Start as if you just installed
it."

## Run Script

### Step 1. Open and orient

Ask:

- "What do you think this app is for?"
- "What do you expect to happen next?"

Record:

- whether the learner understands what the app is
- whether they can infer why it might help them
- whether the first screen motivates a start or feels like friction

### Step 2. Placement

Ask the learner to continue naturally.

Do not help unless they are hard-stuck for more than `20` seconds.

Record:

- hesitation before starting
- confusion around why placement exists
- whether placement feels like useful routing or setup work
- whether any copy sounds abstract, internal, or unmotivating

### Step 3. Home handoff

When `Home` appears, ask:

- "What do you think you should do now?"
- "Why do you think the app sent you here?"

Record:

- whether the main CTA is obvious
- whether the recommendation feels personal and trustworthy
- whether the learner understands the next action without explanation

### Step 4. First useful hand

Ask the learner to take the next action and complete the first useful hand.

After the first result, ask:

- "What did that teach you?"
- "Do you feel more sure about the next decision than before?"

Record:

- whether the learner can explain the teaching point in plain language
- whether the feedback feels specific enough to the spot
- whether an early "aha" actually happened

### Step 5. Return to Home

Ask:

- "If you came back tomorrow, what would you expect to do here?"
- "Would you keep going now or stop? Why?"

Record:

- whether the learner understands the return loop
- whether the product feels worth continuing
- whether motivation dropped because the app never explained itself clearly

## What Counts As A Failure

Count a real product failure when any of these happen:

- learner cannot explain what the app is for
- learner cannot explain why placement is worth doing
- learner reaches `Home` but does not know what action to take next
- first useful hand does not produce a clear teaching point
- learner asks "why am I doing this?" and the route does not answer it
- pre-entry or first-start framing is too weak to motivate starting

Do not count as a product failure:

- isolated simulator noise
- one-off wording preference without confusion
- advanced-feature requests outside the first session

## Evidence Format

For each run, write only these sections:

### Run Header

- participant type
- device
- date

### Route Result

- reached `Placement`: yes/no
- reached `Home`: yes/no
- reached first useful hand: yes/no
- reached early aha: yes/no/unclear

### Confusion Points

- short bullets only

### Trust Breaks

- short bullets only

### Motivation Gaps

- short bullets only

### Quotes

- only short exact quotes that matter

### Verdict

- pass
- pass with one bounded blocker family
- blocked

## Synthesis Rule

After `2-3` runs, collapse findings into one of these outcomes only:

1. `Novice gate closed`
2. `One bounded first-session trust polish family remains`
3. `One bounded pre-entry orientation / first-start framing family remains`
4. `Route is still not clear enough; do not claim product 100`

Do not produce a broad redesign list.

## Stop Rules

1. Stop after `3` runs unless evidence is corrupted.
2. Do not collect more sessions just to confirm a pattern already obvious.
3. If findings split across unrelated owners, stop and classify them rather
   than improvising fixes.
4. If the main blocker is clearly pre-entry explanation, do not hide it under
   generic placement polish.

## Output

The expected output from this protocol is:

- one compact notes file or one compact report section
- one honest gate verdict
- at most one bounded next task family for novice proof residue
