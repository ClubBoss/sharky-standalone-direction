## Guard strategy

- state truth guards
  - street and board-card count alignment
  - preflop board-leak family guards
- hero / actor ownership guards
  - prompt seat, active seat, `heroSeatId`, and hero-marked seat alignment
  - no stale BTN Hero residue in early-seat hero-action tasks
- feedback title / concept-family guards
  - focused guards for reused-runner title drift in W4, W9, and W10
- preflop specificity guards
  - KQo / J8o / A7o facing-open feedback must name the visible cue cleanly
- strategic simplification honesty guards
  - simplified preflop continue binaries with omitted aggressive branches must keep an explicit beginner-baseline scope marker
  - future worlds must pass the same strategic-simplification honesty audit before production-ready status
- duplication / repetition guards
  - exact drill triplet checks use caption, hint, and question together
  - recap, review, transfer, and explicit checkpoint roles stay outside the hard duplicate ceiling
  - reused runner chains should keep new seat, frame, hand, or failure-model value when context changes
  - future worlds must join the same duplication audit before production-ready status
  - prefer learner-facing helper assertions over global internal mirror tests
- future answer-option readability / copy-length guard
  - protect against long option text that wraps poorly or hides the clean action
  - hard-stop explanation-style labels from teaching reasons inside the button
  - future worlds must pass the same answer-option readability audit before production-ready status
- future pot / to-call / legal-option guards
  - verify visible price, action trail, and available actions describe the same spot
  - early-seat unopened runners must keep final trail actor aligned with prompt seat, `activeSeatId`, and hero ownership
- future cognitive-load / anti-bloat guards
  - keep intro teaching-step bodies short enough to teach one idea first
  - watch jargon-heavy teaching steps for local smoothing instead of paragraph growth
  - future worlds must pass the same anti-bloat audit before production-ready status
- first-start route guards
  - keep complete-beginner bypass available before live diagnostics
  - keep first-run intake limited to the admitted `experience -> confidence -> live check` contract unless a new route-contract wave is explicitly admitted
  - keep visible premium/payment framing out of the path before first value
- orientation / terminology guards
  - do not restore long Welcome explanation walls as a shortcut for missing starter-guide architecture
  - keep the first-use orientation layer inside the existing Act0 / World 1 route and outside a canonical `World 0`
  - keep `First Table Guide` as the first normal World 1 lesson; do not add a numeric `Lesson 0` special case
  - keep `First Table Guide` focused on Sharky/product/table literacy; the
    primary Texas Hold'em and cash-first explanation belongs in `What poker is`
  - placement may adapt the poker-content path only after the guide, never before it
  - stronger users may bypass beginner poker-content only after the guide, and those basics must stay replayable without skipped-looking holes
  - do not surface unexplained poker shorthand before the local teaching beat or a deliberate glossary/help affordance exists
- retention spine guards
  - do not treat linear route completion as the same thing as durable mastery
  - keep review / repair / recheck as distinct runtime jobs, not one merged loop
  - add deterministic resurfacing rules before any ML or opaque personalization
  - persisted retention memory is the required substrate for future recheck
    waves; do not fake durable resurfacing with session-only state
  - `Review Memory + Spaced Recheck v1` may use deterministic sequence rules
    and Review-only surfacing before any broader Home training-plan split
  - Home daily-plan waves must keep one primary route action and surface
    repair / recheck / prove as compact secondary routing, not as competing
    hero CTAs
  - world-completion payoff waves must stay compact, describe owned skill in
    learner-facing terms, and emit only a small deterministic set of future
    review targets
  - soft-mastery waves must keep `ownedCandidate` as evidence input, not as a
    fake permanent-mastery claim
  - density-lift waves must add real transfer variants with new board /
    action / price / pressure frames; do not count recap duplication or
    filler duration as retention progress
  - street-change transfer waves must keep one coherent draw story across
    flop / turn / river; do not treat later streets as isolated mini-lessons
  - mistake-review variant waves must map one weak concept to one bounded
    nearby frame; do not flood Review with random substitute drills or replace
    exact replay for every family
  - second-family mistake-review waves should reuse already-landed transfer
    families where possible; keep source-task identity and launched-variant
    identity distinct so the original weak concept remains the repair truth
  - soft daily-plan waves must treat the daily mix as recommended, not
    mandatory; keep one primary route CTA live and avoid hard locks, timers,
    cooldowns, or punitive stop states
  - user-facing pacing should lead with a simple daily training checklist, not
    `30-day plan` or month-obligation framing; long-horizon pacing may exist
    under the hood before it is surfaced directly
  - the first shipped Home expression of that checklist is a premium 4-row
    layout, not the future 2x2 square; portrait clarity and one-primary-CTA
    safety outrank novelty in v1
  - the canonical checklist order is:
    `Learn / Continue`, `Practice / Practice quick spots`,
    `Review old spots`, `Fix mistake / Keep sharp`
  - once the checklist is active, Home should use a compact route strip rather
    than a large route hero / Continue button; the checklist is the main
    activated surface
  - Home row ownership should stay bounded to existing contexts:
    Learn opens Learn, Practice opens Practice, Review opens Review when due,
    and Fix / Keep sharp may use direct repair or prove seams when actionable
  - if no mistake is due, do not show a fake `fix mistakes` duty; collapse the
    block or switch to a clean-state / keep-sharp alternative
  - once the checklist is active, Weekly Focus should render as lightweight
    context above it, not as a standalone obligation card
  - full checklist framing should appear only after the learner has completed
    the first real training loop; before first value, keep Home/Welcome simple
  - weekly-focus waves must stay advisory and lightweight: derive one focus
    from live repair / recheck / current-route data, keep daily jobs primary,
    and do not introduce a separate pacing CTA or persistence layer in v1
  - weekly focus should remain a lightweight context signal beneath the daily
    checklist, not a separate obligation or parallel pacing contract
  - world-end mastery-pack waves must stay compact and evidence-based:
    summarize what the learner can now do plus near-term keep-sharp /
    prove-next direction, but do not claim permanent mastery or add hard
    progression locks
  - future monthly / 30-day layers must not be surfaced as scary obligation
    UX; Home stays daily-first and longer-horizon pathing may later live in
    Profile if runtime backing becomes strong enough
  - retention replay jobs (`agedRecheck`, `ownedCandidate`) must not create
    fake open-repair records when replayed through Review
  - v1 soft mastery must not hard-block route continuation or add punitive
    progression locks
  - future retention waves must prove that repaired concepts can reappear later
    without losing learner trust or creating noisy repetition
  - content expansion alone must not be used as a substitute for pacing,
    mastery, and resurfacing architecture
- future localization / human-copy guards
  - active supported locale slices must not mix EN lesson wrappers with localized task copy on the same learner surface
  - unsupported worlds may fall back cleanly, but the fallback boundary must stay explicit in coverage tests
  - future worlds and future locales must pass the same mixed-locale and fallback-coverage audit before production-ready status
- full-route novice-proof guards
  - do not represent Act0 as `100 / 100` until novice comprehension, first
    value, Review naturalness, and "got better today" proof are recorded
  - Home should stay daily-first and closed as a composition target unless a
    new concrete blocker appears
  - Profile / You may later own longer-horizon pathing, but Home must not
    surface scary `30-day` or monthly obligation framing in the current route
- prompt / context guards
  - decision prompts must not invent facing context when no actor source is
    safe
  - when a prior action context is expected, prove the actual active context
    line instead of relying on stale runner inheritance
