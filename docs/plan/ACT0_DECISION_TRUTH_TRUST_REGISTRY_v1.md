## Guard strategy

- state truth guards
  - street and board-card count alignment
  - preflop board-leak family guards
- hero / actor ownership guards
  - prompt seat, active seat, `heroSeatId`, and hero-marked seat alignment
  - no stale BTN Hero residue in early-seat hero-action tasks
- feedback title / concept-family guards
  - focused guards for reused-runner title drift in W4, W9, and W10
- preflop specificity guards
  - KQo / J8o / A7o facing-open feedback must name the visible cue cleanly
- strategic simplification honesty guards
  - simplified preflop continue binaries with omitted aggressive branches must keep an explicit beginner-baseline scope marker
  - future worlds must pass the same strategic-simplification honesty audit before production-ready status
- duplication / repetition guards
  - exact drill triplet checks use caption, hint, and question together
  - recap, review, transfer, and explicit checkpoint roles stay outside the hard duplicate ceiling
  - reused runner chains should keep new seat, frame, hand, or failure-model value when context changes
  - future worlds must join the same duplication audit before production-ready status
  - prefer learner-facing helper assertions over global internal mirror tests
- future answer-option readability / copy-length guard
  - protect against long option text that wraps poorly or hides the clean action
  - hard-stop explanation-style labels from teaching reasons inside the button
  - future worlds must pass the same answer-option readability audit before production-ready status
- future pot / to-call / legal-option guards
  - verify visible price, action trail, and available actions describe the same spot
  - early-seat unopened runners must keep final trail actor aligned with prompt seat, `activeSeatId`, and hero ownership
- future cognitive-load / anti-bloat guards
  - keep intro teaching-step bodies short enough to teach one idea first
  - watch jargon-heavy teaching steps for local smoothing instead of paragraph growth
  - future worlds must pass the same anti-bloat audit before production-ready status
- future localization / human-copy guards
  - active supported locale slices must not mix EN lesson wrappers with localized task copy on the same learner surface
  - unsupported worlds may fall back cleanly, but the fallback boundary must stay explicit in coverage tests
  - future worlds and future locales must pass the same mixed-locale and fallback-coverage audit before production-ready status
