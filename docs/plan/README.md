# Active Plan Bucket

`docs/plan/` is reserved for active planning truth:
- master plans
- product-working authority and route order
- launch-surface route contracts
- active curriculum and retention SSOTs
- execution maps
- readiness references and pacing rules

Start here when the task is about the current product:

1. `docs/plan/PROJECT_TOPOLOGY_AND_TRUTH_MAP_v1.md`
2. `docs/plan/MASTER_PLAN_v3.0.md`
3. `docs/plan/LAUNCH_SURFACE_MECHANISM_v1.md`

Use `docs/plan/PROJECT_READINESS_EPICS_SSOT_v1.md` only when launch/readiness
framing is explicitly needed.

This bucket is not the default home for seam audits or closeout packets.
Historical audits and older execution traces belong under `docs/archive/` or
their dedicated audit/governance buckets.
