# Volume I World Quality Scorecard v1

Status: ACTIVE
Last updated: 2026-05-06

## Purpose

Provide one compact calibration frame for `W1-W12` so world quality is judged
consistently instead of by local intuition.

This is not a new curriculum SSOT.
It is a reusable evaluation frame for the launch-critical first volume.

## Governing Inputs

Use this scorecard with:

1. `docs/plan/MASTER_PLAN_v3.0.md`
2. `docs/content/CONTENT_PLAN_PER_WORLD_v2.1.md`
3. `docs/content/CONTENT_EXCELLENCE_CANON_v1.md`
4. `docs/plan/RETENTION_RHYTHM_ANTI_BOREDOM_v1.md`
5. `docs/content/CONTENT_QA_PACKET_v1.md`
6. relevant seam audit artifacts

## Core Rule

Do not optimize for equal lesson counts.

Optimize for equal felt completeness:

1. each world clearly teaches its intended job
2. each world has enough density to feel finished for that job
3. each world has enough variation to avoid grind without harming stability
4. each world hands off cleanly into the next one

## Scoring Axes

Score each axis as:

- `3 = strong`
- `2 = adequate`
- `1 = thin`
- `0 = missing or misleading`

### 1. Concept Clarity

Question:

- does this world install one dominant cognitive shift, or is it mixed and
  blurry?

Strong means:

- one primary learner job
- supporting ideas reinforce that job instead of competing with it

### 2. Density Completeness

Question:

- does this world feel structurally complete for its job?

Strong means:

- Intro / Practice / Apply / Review rhythm is real
- the world does not feel like a teaser, shell, or thin sketch

### 3. Decision Richness

Question:

- does the world give enough meaningful reps to convert explanation into habit?

Strong means:

- the learner makes enough real decisions, taps, compares, or recalls
- the world is not mostly passive explanation

### 4. Variety And Drill Mix

Question:

- does the world vary the interaction shape enough to stay interesting while
  preserving the same main job?

Strong means:

- the world uses bounded variation rather than one repeated prompt loop
- drill-family use supports learning, not novelty for novelty's sake

### 5. Repair And Review Quality

Question:

- does the world contain a predictable mistake model and a clean repair path?

Strong means:

- wrong answers teach something useful
- recap/review reinforce the right habit without shame

### 6. Transfer Value

Question:

- does the world create a believable carryover into real play or real table
  observation?

Strong means:

- the learner can notice or do something outside the app because of this world

### 7. Seam And Handoff Quality

Question:

- does this world bridge safely into the next one?

Strong means:

- vocabulary handoff is clean
- the next world feels like progression, not a topic jump

## Overall Bands

Use these bands after scoring the 7 axes:

- `19-21`: release-strong
- `16-18`: solid but still polishable
- `12-15`: adequate, but visible thinness remains
- `8-11`: fragile
- `0-7`: not ready to represent the route

## Public Launch Bar For Volume I

For internal build work, `16-18` is acceptable while a visible world is still
being strengthened.

For public `Volume I` launch quality, the target is stricter:

1. every visible `W1-W12` world should target at least `19`
2. `21` is aspirational, not mandatory
3. any visible world still below `19` is not yet at the intended public
   release-strength bar

This keeps the launch route honest:

- not every world must be maximally saturated
- but no public world should still feel obviously second-tier beside the others

## Operational Use

Use this scorecard when:

1. calibrating the current state of `W1-W12`
2. deciding which world needs the next density packet
3. checking whether a new drill-family packet improved the route materially

Do not use it to justify endless polishing after a world is already clearly in
the `release-strong` or `solid but still polishable` bands.

## Tie-Break Rule

If two candidate worlds both need work, prioritize:

1. the world with lower Seam And Handoff Quality
2. then the world with lower Concept Clarity
3. then the world with lower Density Completeness

This keeps route integrity above cosmetic variety work.
