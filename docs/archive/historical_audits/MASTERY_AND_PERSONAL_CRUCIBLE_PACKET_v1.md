# Mastery And Personal Crucible Packet v1

Status: ACTIVE
Last updated: 2026-05-06

## Purpose

Turn the existing three-layer depth idea into one execution-ready content
system packet:

1. finite core path
2. deterministic mastery tiers
3. personal weak-spot resurfacing

The goal is not infinite content.
The goal is durable depth, strong return motivation, and future subscription
value without bloating the course.

## Core Principle

Depth should come from:

1. better reuse of strong atoms
2. tighter proof of competence
3. resurfacing the learner's actual weak spots
4. stronger identity and progress signals

Depth should not come from:

1. endless new lessons
2. noisy randomness
3. fake grind
4. pressure without felt improvement

## The Three Layers

### Layer A. Core Path

What it is:

- the finite curated route
- worlds that introduce the main cognitive shifts

Job:

- teach the concept clearly
- stabilize the first useful behavior
- create one small real-play transfer seam

### Layer B. Mastery Tiers

What it is:

- the same atom reused vertically after first completion

Canonical tiers:

1. `Learn`
   - hints allowed
   - clean examples
   - standard pace
2. `Prove`
   - fewer hints
   - tighter thresholds
   - more mixed recall
3. `Speed`
   - low guidance
   - faster decisions
   - mixed contexts

Job:

- convert "I saw this once" into stable competence
- make the learner feel sharper over time
- extend value without building a second full curriculum

### Layer C. Personal Crucible

What it is:

- deterministic resurfacing of the learner's weak families
- not AI generation
- not a free-form coach

Job:

- protect the learner from forgetting what matters
- make weakness feel repairable, not punishing
- create the strongest reason to return because the app feels personally useful

## Return And Subscription Relationship

This packet is one of the main long-horizon habit engines in the product.

Why:

1. core path creates the first wins
2. mastery tiers create felt sharpening
3. personal crucible makes return sessions personally relevant
4. that builds durable trust and habit
5. later paid depth feels like real expansion, not rescue

Best long-term loop:

learn -> notice improvement -> prove it -> repair weakness -> return

This is stronger than:

1. raw streak pressure
2. empty reward inflation
3. constant novelty without mastery

## Mastery Tier Rules

### What may change across tiers

Allowed:

1. hint level
2. pacing
3. recall mix
4. context variation inside the same family
5. threshold strictness

Not allowed:

1. brand-new prerequisite concepts disguised as higher tier
2. full route reorder
3. random unbounded drill mixing
4. solver-language leakage into beginner families

### Promotion Rule

An atom should move upward only when the learner has shown stable evidence, not
one lucky result.

Promotion should come from:

1. repeated correctness
2. stable same-family performance
3. checkpoint or capstone proof where relevant

### Learner Feeling Rule

The learner should feel:

- "same skill, cleaner and faster now"

Not:

- "the app is just making me repeat old stuff"

## Personal Crucible Rules

### Canonical Input

Personal crucible should pull from:

1. leak severity
2. repeated misses
3. unstable family signals
4. resurfacing due logic
5. recent repair history

### Canonical Output Shapes

1. `Replay this spot`
2. `Same family, easier rep`
3. `Same family, mixed rep`
4. `Repair before forward progress`
5. `Weak spot returns today`

### Personal-Crucible Tone

The system must feel:

1. useful
2. calm
3. earned
4. specific

It must not feel:

1. punishing
2. guilt-heavy
3. random
4. like the app is scolding the learner

## Where Each Layer Lives

| Layer | Primary surface job | Secondary surface job |
| --- | --- | --- |
| Core Path | Learn | Review and transfer |
| Mastery Tiers | Play / mastery replay | Home and Profile progress proof |
| Personal Crucible | Review / best-next-action | Home daily return and weak-spot routing |

## Daily Habit Relationship

The daily loop should feel fresh because of controlled depth, not because of
chaos.

Good daily-return mix:

1. one clear best-next action
2. occasional mastery-proof rep
3. occasional weak-spot resurfacing
4. visible evidence of sharpening

Bad daily-return mix:

1. endless same local lesson
2. unpredictable soup
3. pressure-first streak guilt
4. empty reward loop with no learning difference

## Retention Hierarchy

When choosing what drives return, use this order:

1. real usefulness
2. visible improvement
3. personal relevance
4. compact celebration
5. streak-lite support

Do not invert this hierarchy.

## Production Rules

Before creating new content for depth, prefer this order:

1. reuse an existing atom in `Prove`
2. reuse it again in `Speed`
3. create a personal-crucible resurfacing shape
4. only then consider whether new authored content is actually needed

This is how the product avoids infinite-content debt.

## Operational Questions

Before saying a family has durable long-term depth, ask:

1. does it have a clear core-path version?
2. can the same atom be reused across `Learn / Prove / Speed`?
3. can weak performance surface a calm personal-crucible repair?
4. does this make return feel useful tomorrow?
5. does this increase future paid depth without requiring infinite new lessons?

If those answers are weak, the family is not yet ready to carry long-term
retention value.

## Immediate Implication

The next active content-governance standard should be:

1. worlds teach the first win
2. mastery tiers prove sharpening
3. personal crucible makes return personal

That is the most honest route to a product people keep coming back to for a
long time.
