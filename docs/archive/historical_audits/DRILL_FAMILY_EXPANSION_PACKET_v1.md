# Drill Family Expansion Packet v1

Status: ACTIVE
Last updated: 2026-05-06

## Purpose

Define the next bounded drill-family expansion for `Volume I` and the immediate
post-core bridge without creating new product modes, new screens, or old-main
host sprawl.

This packet preserves the strongest old-main exercise ideas in a compact form:

1. `bet_sizing_choice`
2. `showdown_compare`
3. `counting_choice`

## Why These Three

These families are the highest-EV next step because they strengthen the first
12 worlds directly:

1. `bet_sizing_choice`
   - sharpens W5 and later postflop worlds
   - teaches size-to-purpose, not random button tapping
2. `showdown_compare`
   - sharpens W1, W6, and later hand-strength worlds
   - prevents weak absolute-strength understanding
3. `counting_choice`
   - sharpens W5-W8 and later postflop/math worlds
   - gives one reusable family for outs, combos, and simple price counts

## Non-Goals

This packet does not authorize:

1. new standalone learner modes
2. a second runner architecture
3. early Cash vs MTT branching
4. solver-style math surfacing in the first volume

## Family Order

Use this rollout order:

1. `bet_sizing_choice`
2. `showdown_compare`
3. `counting_choice`

Reason:

- sizing is the cleanest practical bridge from current action-choice work
- showdown compare is highly intuitive and table-adjacent
- counting is powerful, but easiest to overcomplicate if introduced first

## Family Contracts

### 1. bet_sizing_choice

Cognitive job:

- choose the best size from a bounded legal set
- connect size to immediate purpose

Best first homes:

1. W5 `Bet Purpose And Price`
2. later W13-W16 postflop depth

Minimal deterministic truth:

- authored legal size ids
- authored expected size id
- authored local purpose
- authored correct and incorrect feedback

### 2. showdown_compare

Cognitive job:

- decide which visible holding wins at showdown
- build relative hand strength literacy

Best first homes:

1. W1 recap and repair
2. W6 board-and-draw contrast work
3. later W13 relative-strength worlds

Minimal deterministic truth:

- authored player holdings
- authored board state
- authored expected winner id or tie result
- authored correct and incorrect feedback

### 3. counting_choice

Cognitive job:

- count a bounded factual quantity without opening full math mode

Allowed early uses:

1. simple outs
2. simple combo counts
3. simple price or stack count anchors

Best first homes:

1. W5-W8
2. later W13+ postflop and range depth

Minimal deterministic truth:

- authored bounded numeric answer set
- authored expected count
- authored local object being counted
- authored correct and incorrect feedback

## Early-Volume Safety Rules

Inside `W1-W12`, these families must stay beginner-safe.

That means:

1. no solver jargon
2. no formula-dump presentation
3. no large free-response math
4. no hidden-context trick questions
5. no format-specific policy claims masquerading as universal truth

## Runner Rule

All three families should live inside the current compact runner.

Do not build:

1. a separate math mode
2. a separate showdown applet family
3. a separate sizing-only product track

The family is a task type, not a new product branch.

## Volume I Recommendation

For the "make the first 12 worlds ideal" route:

1. keep `W13+` as honest `coming soon`
2. strengthen `W1-W12` with richer task families only where they clearly
   sharpen the current concept atom
3. prefer one added family at a time over broad simultaneous family rollout

## Next Implementation Recommendation

If one implementation wave is admitted next, the best first packet is:

1. expand `DRILL_CONTRACT_v1.md`
2. add runtime/model support for `bet_sizing_choice`
3. add narrow tests

That gives the highest immediate learner value without broadening scope.
