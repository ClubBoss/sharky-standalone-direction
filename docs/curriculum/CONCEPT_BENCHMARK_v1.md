32. Что особенно важно не упустить в приложении

Самые часто недооцененные, но EV-critical зоны:

1. Позиции на визуальном столе, не только текстом UTG/CO/BTN.
2. Кто ходит первым / последним на каждой улице.
3. Board reading до любых сложных решений.
4. Kicker logic, потому что новички часто думают “у нас обоих пара, значит ничья”.
5. Clean vs dirty outs, потому что простое “у меня 9 outs” часто ложное.
6. Relative hand strength, потому что top pair не всегда сильная рука.
7. Position-driven decision changes: та же рука IP и OOP играется иначе.
8. Stack depth / SPR, потому что без этого игрок не понимает commitment.
9. Preflop discipline, потому что большинство beginner EV сгорает до флопа.
10. Value betting, потому что слабые игроки больше теряют не на блефах, а на missed value.
11. River underbluff population exploit, особенно для live/low-stakes.
12. Multiway discipline, потому что новички часто переигрывают одну пару в multiway.
13. Mistake recovery loop, не просто “wrong”, а “почему wrong → повторить похожий spot”.
14. Bankroll and tilt, потому что без этого даже технически хороший игрок ломается.
15. Study workflow, потому что путь до pro невозможен без review system.

34. Главный вывод для твоего приложения

Самая сильная архитектура curriculum для Poker Analyzer — не просто “список уроков”, а skill graph:

Rules
  -> Table literacy
  -> Position literacy
  -> Hand reading
  -> Board reading
  -> Preflop discipline
  -> Pot odds / outs
  -> Postflop reasons
  -> Range thinking
  -> Bet sizing
  -> Exploit/GTO split
  -> Format specialization
  -> Pro workflow

И каждый concept должен иметь не только текст, а минимум 4 слоя:

Understand -> Recognize -> Decide -> Recover from mistake

Для максимального EV в твоем формате я бы особенно усиливал:

visual table simulation
+ guided seat/position learning
+ board-state-first reading
+ mistake replay
+ range/equity intuition
+ real decision drills
+ adaptive leak correction

То есть игрок должен не “читать про покер”, а видеть стол, понимать ситуацию, принимать решение, ошибаться безопасно, получать объяснение и тут же переигрывать похожий spot. Это и есть путь от новичка к профессиональному decision engine.
