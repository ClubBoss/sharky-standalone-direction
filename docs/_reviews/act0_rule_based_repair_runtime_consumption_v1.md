# Act0 Rule-Based Repair Decision Runtime Consumption v1

Date: 2026-06-19
Branch: `codex/act0-rule-based-repair-runtime-consumption-v1`
Base commit: `49be286`
Mode: bounded deterministic runtime consumption

## 1. Product Goal

Consume `Act0RuleBasedRepairDecisionV1` inside the existing Act0 repair runtime
path so the deterministic rule-based decision is available to runtime state
after a missed choice.

The user-facing promise stays unchanged:

- the learner missed a visible clue;
- the next useful hand repairs the same clue or exact replay;
- the reason is deterministic and auditable.

This wave does not add public AI, ML, solver, GTO, paywall, trial, pricing,
Premium Hub, or commerce behavior.

## 2. Runtime Seam Used

Runtime consumption uses the existing private next-useful-hand reason receipt:

- `_nextUsefulHandReasonReceiptV1(...)`
- `_Act0NextUsefulHandReasonReceiptV1.forRepairIntent(...)`
- `_openRepairIntentTargetForSourceTaskV1(...)`

That seam already owns the internal bridge from:

- open repair intent;
- launchable mapped target or exact replay fallback;
- selection source;
- missed signal;
- skill atom;
- mapping type;
- reason code;
- next-useful-hand debug payload;
- Review support copy snapshot.

No new route owner, persistence owner, telemetry owner, or UI owner was added.

## 3. Contracts Changed / Unchanged

Changed:

- `buildAct0RuleBasedRepairDecisionV1(...)` now accepts optional resolved
  runtime target overrides:
  - `resolvedTargetWorldId`
  - `resolvedTargetLessonId`
  - `resolvedTargetTaskId`
  - `resolvedMappingType`
  - `resolvedReasonCode`
- `_Act0NextUsefulHandReasonReceiptV1` now carries an optional
  `repairDecision` when the receipt is built from an open repair intent.

Unchanged:

- `Act0RepairIntentV1` payload shape.
- Repair intent creation, retention, clearing, and fallback lifecycle.
- Existing selection sources:
  - `repair_intent_mapped`
  - `repair_intent_exact_replay`
  - `existing_fallback`
- Existing copy guard templates.
- Existing telemetry event names and emission ownership.

## 4. Public UI / Copy Status

No public UI or copy changed.

The existing Review support copy and next useful hand behavior continue to use
the established copy bridge and safe templates. The new decision is carried in
the private/debug receipt payload only.

## 5. Telemetry Ownership Status

Telemetry ownership remains unchanged.

Existing local telemetry still belongs to the current runner/shell seams:

- `task_result` carries `choiceId`, `result`, and `errorType`.
- Feedback and repair events stay local through `Act0TelemetrySinkV1`.
- `ACT0_TELEMETRY_TRUTH_MAP_v1.md` still records `user_choice` and
  `decisionTimeBucket` / `time_to_decision` as future event contract work.

This wave does not add telemetry events, network telemetry, dashboards, vendor
SDKs, or telemetry-owned personalization state.

## 6. Determinism Guarantees

The consumed decision is deterministic:

- no time;
- no RNG;
- no network;
- no persistence;
- no external state;
- no ML;
- no new dependency.

Decision input is limited to:

- the existing open repair intent;
- whether the intent is still open;
- the resolved launchable target already selected by Act0;
- existing mistake attempt count for repeat priority.

Exact fallback is resolved at runtime by passing the selected exact replay target
into the decision builder. This prevents a stored mapped intent from reporting a
mapped decision when the runtime has safely fallen back to exact replay.
