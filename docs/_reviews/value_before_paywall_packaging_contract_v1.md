# Value-Before-Paywall Packaging Contract v1

## 1. Verdict

`packaging_contract_ready_with_copy_recommendation`

This is a docs-only future-packaging contract. It authorizes neither a paywall
nor a pricing, trial, entitlement, subscription, purchase, restore, route, or
content change.

## 2. Current monetization truth

- No active hard paywall exists on the canonical Act0 learner route.
- No active premium world unlock, pricing claim, purchase, restore, trial
  start, or entitlement change exists on that route.
- A completed-value soft preview may exist as a boundary-neutral explanation:
  `Practice extra reps` remains dominant and `See what premium adds` is
  secondary. It does not transact, gate the route, or name world access.
- `W1-W4` remains the strategic free public foundation and `W5+` remains a
  future paid-depth boundary; that boundary is strategic policy, not current
  paid access behavior.
- W13+ is not unlockable. W11-W12 is planned only.
- Older premium, trial, IAP, and entitlement families remain in the repository
  as historical/dormant implementation context. They do not override the
  launch-facing `MONETIZATION_SSOT_v1` or authorize current Act0 commerce.

## 3. Value-before-paywall principle

Before any future paid prompt, a learner must have experienced a complete free
learning loop: a real table decision, clear feedback tied to a visible signal,
and a useful next step or repair. The first-value W1-W4 foundation must remain
trust-building rather than being a teaser that withholds basic understanding.

Any future ask must follow visible learning proof, not placement, first hand,
first feedback, first completion, or the soft preview itself. Paid depth can
extend already-earned usefulness; it cannot rescue a blocked route, substitute
for content proof, or pressure the learner with urgency.

## 4. Safe future packaging model

| Bucket | Allowed label and wording | Forbidden wording | Proof before upgrade |
| --- | --- | --- | --- |
| Free/current route-backed foundation | `Current foundation`; `Continue the current route.` | `Free trial required`, `Upgrade to continue`. | Current route, learner value, and focused proof remain green. |
| Planned foundation | `Planned foundation`; `Coming later.` | `Locked`, `Premium`, `Open W11 now`. | Active shelf, canonical route ownership, learner entry, and route proof. |
| Later paid-depth candidate | `More depth is planned after the free foundation.` | `Paid depth available now`, named-world premium access, price, trial, or purchase copy. | W5+ scope/content proof, route/access contract, commerce safety, and admitted packaging decision. |
| Later strategic frontier | `Later strategic depth.` | `Unlock W13`, `Upgrade for W13`, `Specialization ready`. | Authored route, learner entry, focused route proof, and revised status contract. |
| Advanced specialization frontier | `Later advanced frontier.` | `Full Cash/MTT tracks`, `Become pro`, specialist access. | Explicit specialization policy plus owned content, route, readiness, and access proof. |

## 5. Proof gates before paid claims

| Claim | Required gate |
| --- | --- |
| W5+ paid depth | Stable W5+ learner content and route truth; explicit free/paid boundary decision; production commerce safety; non-coercive value proof. |
| W11-W12 | Active-root shelf, canonical session/campaign registration, learner entry, and focused passing route proof. |
| W13+ unlock/access | The W11-W12 gate plus owned W13+ content, route/access contract, and revised status truth. |
| Specialization | Explicit specialization policy, real Cash/MTT route/content proof, and readiness evidence. |
| AI/adaptive | Owned adaptive behavior visible to learners and proof that it changes recommendations safely. |
| Mastery/leak | Owned mastery or leak-resurfacing behavior, persistence, learner-facing proof, and acceptance tests. |
| Cash/MTT track | Named active route, current content, learner entry, and test evidence; W10 labels alone are insufficient. |
| App Store/external commercial claim | Real external recipient, current screenshot proof, commerce safety, and legal/store readiness. |

## 6. Forbidden monetization/copy list

Do not use these phrases or close variants on current learner surfaces:

- `Unlock W13`
- `Upgrade to continue`
- `Premium specialization`
- `Start your trial to keep learning`
- `Full Cash/MTT tracks`
- `AI fixes your leaks`
- `Mastery path live`
- `Become pro`
- `Complete Volume I now`
- `Paid depth available now`
- price, countdown, scarcity, guilt, expiry, or pressure wording that makes a
  learner fear losing basic usefulness.

## 7. Future allowed copy examples

- `More depth is planned after the free foundation.`
- `The current route continues through W10.`
- `W11-W12 are planned foundation chapters, coming later.`
- `Later depth may become part of paid learning when it is ready.`
- `No purchase is needed for the current route.`
- `If deeper learning is offered later, show the value before asking for an
  upgrade.`

These are future-contract examples, not current prompts or authorization to
expose commerce.

## 8. Surface placement rules

| Surface | Rule |
| --- | --- |
| Learn | Explain current route and later landmarks. Do not show price, locked-depth access, trial, or purchase controls. |
| Home | Keep the current learning action dominant. Do not add premium dispatcher, countdown, or future-world CTA. |
| Profile | May mirror current progress only; never imply paid identity, mastery, or specialist access. |
| Review | Keep repair value free and useful; never make correction contingent on payment. |
| Future paywall | Only after the proof gates above and a separately admitted implementation contract. |
| App Store/external packaging | Deferred until a real external recipient and a separately admitted readiness audit exist. |
| Screenshot proof packets | Show real current value only; never insert fake monetization for packaging. |

## 9. Risk ledger

| Risk | Guard |
| --- | --- |
| Trust break | Earn a visible learning win before any future ask. |
| Fake unlock | Never name W11+ or W13 access before route/access proof. |
| Premature paywall | Preserve the W1-W4 free trust route and non-commercial soft preview boundary. |
| Route mismatch | Treat current campaign truth and access truth as separate from planning labels. |
| Paid-depth without content proof | Require authored content, route, and learner proof before access language. |
| Runout-clone pressure | Sell demonstrated sharpening, not library breadth, urgency, or opaque analytics. |
| Conversion over learning trust | Keep repair, feedback, and current continuation useful without payment. |
