# Recap

## Key Ideas
- Capture atomic notes during play (action, texture, size, result) and route hands to fix_now, study_spot, or player_read for follow-up.
- Tags focus on stable traits and pool reads so repeatable patterns emerge without stealing table focus.

## Common Mistakes
- Writing essays mid-session, which causes missed cards and tilts the rhythm.
- Using vague labels like "spewy," which cannot be searched or applied in future reviews.

## Quick Checks
- What does a quality note include, and where does it go afterward?
- How do tags like overfold_flop or calls_down earn value across sessions?

## Mini-Exercise (1–2 lines)
- Capture a K72r flop where you call small and later fold; note the tag and which study bucket you send it to.
