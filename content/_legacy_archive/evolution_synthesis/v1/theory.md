# What it is
Synthesis is the unified decision model that combines range basics, layering, advantage, weighting, exploit geometry, pressure mapping, reaction layers, and exploit windows. It describes how these modules work together so every street, node, and adjustment follows the same structural logic instead of pulling apart into fragmented strategies.

# Why it matters
When synthesis governs the table, you get stable scalability: no more conflicting instincts, consistent exploit calibration, and systematic reactions to pressure moves. A synthesis mindset eliminates gaps between concepts, lets you see how advantage feeds weighted aggression, and keeps exploit pacing matched to opponent tendencies.

# Rules of thumb
- Synthesis begins with range structure, not exploits, so value bands anchor every decision.
- Pressure plus reaction define feasible actions before you add bluffs or thin value.
- Exploit windows overlay onto stable geometry; do not chase them without a structural map.
- Collapse zones reshape bluff and value bands, so adjust weights before firing lines.
- Density shifts govern pressure scaling; density maps must lead the weighting.
- Weighting stabilizes aggression across nodes and keeps windows from destabilizing the range.

# Mini example
Read the pressure tiers, observe a reaction-layer collapse, identify a short-lived exploit window, and then apply conditional aggression whose weights align with the underlying range. Every step references the same synthesis map: structure first, then pressure, then windowed exploitation.

# Common mistakes
- Treating the modules as independent rather than integrated pieces of the same model.
- Applying exploit windows without the pressure context that validates them.
- Ignoring reaction layers when adjusting aggression, leaving gaps.
- Separating weighting from geometry and letting biases dominate.
- Misreading collapse zones as raw advantage and forcing action.
- Neglecting density when choosing lines, so synthesis loses balance.

# Workflow
- Assess your core range structure to understand where value lives.
- Map advantage plus density to identify where leverage already exists.
- Identify pressure tiers so aggression matches the opponent’s capability.
- Classify reaction layers to know how the range responds to pressure.
- Detect exploit windows within that architecture.
- Assign weighted aggression across the relevant nodes.
- Verify stability by checking coverage and adjust to opponent tendencies.

# Mini-glossary
- synthesis : the unified framework that ties all modules into one decision model.
- integration layer : the mental layer that aligns ranges, pressure, and reactions.
- density chain : the progression of combo density across pressure tiers.
- collapse-response loop : the feedback cycle that defines how ranges shrink under pressure.
- exploit alignment : timing aggression with structural openings.
- pressure lattice : the grid of tiers where pressure operates.
- reaction mapping : tracking how a range flexes under stress.
- window sequencing : the order in which exploit windows open and close.
- stabilizer flow : the combos that defend when everything else shifts.
- unified weighting : the consistent assignment of frequencies across streets.

# EV
Full synthesis reduces fragmentation, increases stability, and turns multi-street plans into reliable sequences. When every window, reaction, and pressure tier leads back to the same map, your EV stays positive because you no longer chase isolated concepts but follow an integrated exploit path.
