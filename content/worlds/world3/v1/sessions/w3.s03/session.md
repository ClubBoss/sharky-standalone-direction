# Session w3.s03

## Objective
Consolidate the first preflop framework through a one-reason mixed checkpoint.

## Scenario
You continue in bounded preflop spots. Each rep asks for one clear first action based on hand category and light context, so the framework becomes stable before deeper sizing, board-texture, range, or specialization layers appear.

## Decision
Read the hand category first, then choose the cleanest preflop action: open, call, or fold.

## Explanation
This is the one-reason mixed checkpoint for the World 3 front slice. Repeating simple hand-bucket decisions helps the learner build a usable preflop map before the curriculum asks for richer judgment later.
